SME Library MCP — Implementation Checklist

What it takes to stand up and run the Subject Matter Expert MCP service · Vercel + Supabase

This is the deployment and operations runbook for the SME Library MCP server. Items marked code done are implemented in the repository; items marked action are steps you perform to get a live service. Work top to bottom.

Architecture in one line: a Next.js app on Vercel serves MCP over Streamable HTTP at /api/mcp, backed by a standalone Supabase project (Postgres + Auth + pgvector + Edge Functions). Agents authenticate with hashed bearer API keys; humans use a Supabase-Auth dashboard to mint keys.

1 · Prerequisites

  • A Supabase account and a new, dedicated project (standalone — not shared with other apps)
  • A Vercel account connected to the GitHub repo
  • An Anthropic API key (used by the generate_sme tool)
  • The Supabase CLI installed locally (npm i -g supabase) and supabase login done
  • Node.js 18+ locally for the bootstrap script and optional local runs

2 · Supabase — database

  • Create the project; note the project ref, URL, anon key, and service_role key
  • supabase link --project-ref <ref>
  • Apply all migrations: supabase db push code done (migrations 001–003 in repo)
  • Confirm the vector and pgcrypto extensions enabled (migration 001 does this)
  • Confirm the seeded library workspace row exists (id 0000…0001)
  • Verify RLS is enabled deny-all on every table (migration 001) and the authenticated read policies exist (migration 003)

3 · Supabase — Auth & Edge Functions

  • Enable Email auth provider (Authentication → Providers). Decide whether to require email confirmation
  • Add your Vercel domain to Auth → URL Configuration → Redirect URLs (https://<domain>/auth/callback)
  • Deploy the embedding function: supabase functions deploy embed code done (gte-small, 384-dim) — optional; search falls back to keyword FTS if absent

5 · Deploy

  • Import the repo into Vercel (framework auto-detected as Next.js)
  • vercel deploy --prod (or push to the production branch)
  • Confirm the daily cron is registered from vercel.json code done (Settings → Cron Jobs)
  • Hit GET /api/health — expect configured.supabase and configured.anthropic both true

6 · First keys & smoke test

  • Mint an operator/admin key: node scripts/create-key.mjs --workspace "Ops" --admin code done
  • Or self-serve: visit /dashboard, sign up, create a read/write key
  • Connect a client:
    claude mcp add --transport http sme-library https://<domain>/api/mcp --header "Authorization: Bearer sme_live_…"
  • Exercise each tool: create_sme → search_smes → get_sme → update_sme → generate_sme → record_feedback → propose_promotion → clone_sme → archive_sme
  • As an admin, approve a proposal at /dashboard/admin; confirm it appears in GET /api/library
  • Confirm auth is enforced: unauthenticated /api/mcp and /api/cron/maintenance return 401

8 · Remaining engineering work action

  • OAuth 2.1 (dynamic client registration + PKCE) so claude.ai custom connectors authorize interactively instead of pasting a key — mcp-handler supports it via withMcpAuth; needs Supabase configured as an OAuth authorization server
  • Stale-SME decay: flag SMEs unused for N months / declining scores for deprecation review (extend the cron)
  • LLM content screen on promotion: auto-reject personas designed for harm before they enter the library
  • Usage dashboards: per-workspace metering views over audit_log
  • report_sme path for community moderation of library entries
  • Paid tiers: wire rate_limit_tier to billing; higher quotas, private-workspace limits
  • Automated tests + CI: unit tests for auth/search/promotions; GitHub Actions on PR

9 · Operations

  • Verify the maintenance cron runs (Vercel → Cron logs); it recomputes quality, prunes rate-limit windows, backfills embeddings
  • Rotate leaked keys via the dashboard (revoke) or DB; keys are hashed so plaintext can't be recovered
  • Monitor audit_log for anomalous write volume per key
  • Back up the Supabase database (point-in-time recovery on paid plans)
  • Watch Anthropic spend — generate_sme is quota-limited per key but bills to your account
